## 📝 Documentation Standards

### File Naming
- Use UPPERCASE for documentation files
- Use underscores for multi-word names
- Use .md for markdown, .txt for plain text

### Content Structure
- Start with clear title
- Include table of contents for long docs
- Use headers for organization
- Include code examples where relevant
- Add visual diagrams when helpful

### Markdown Style
- Use `code blocks` for commands
- Use **bold** for emphasis
- Use *italics* for notes
- Use > blockquotes for important info
- Use tables for structured data
- Use lists for sequential items
